# Temario (17 semanas)

Cada semana incluye: objetivo, práctica, **benchmarks** y criterios de aceptación.

## Semanas
1. Arquitectura CPS y setup de repositorio (diagrama + proyecto)
2. Embebidos I: GPIO + sensado real (incluye mediciones)
3. Embebidos II: buses I2C/SPI/UART
4. Interfaz local 1: USB/Serial + protocolo simple (CSV/JSON/CBOR)
5. Interfaz local 2: Bluetooth (BLE o clásico) o WiFi local
6. Redes: TCP/UDP + latencia (stress test)
7. MQTT: pub/sub, QoS, topics, retención
8. Servidor local (Python): gateway + API REST
9. Web realtime: WebSocket o SSE
10. Nube 1: despliegue (Render/VPS) + API pública
11. Frontend JS: dashboard remoto + control (HTTP)
12. Persistencia: base de datos ligera + series de tiempo (incluye Firebase)
13. IA 1: visión o clasificación (local o nube)
14. IA 2: LLM como capa de decisión segura (auditable)
15. Integración industrial: Modbus u OPC-UA (puente CPS ↔ industrial)
16. Integración XR (opcional) o robustez/seguridad (hardening)
17. Demo final + evaluación

> Recomendación didáctica: el **proyecto final** se construye incrementalmente.
